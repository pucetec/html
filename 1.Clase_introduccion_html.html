🌐 ¿Cómo funciona un website?

Un website funciona gracias a la comunicación entre un navegador y un servidor. El navegador solicita recursos, el servidor responde y el navegador interpreta esos archivos para mostrar la página.

Flujo básico

  1. El usuario escribe una URL o hace clic en un enlace.
  2. El navegador envía una solicitud HTTP al servidor.
  3. El servidor responde con archivos como HTML, CSS, JavaScript, imágenes o datos.
  4. El navegador procesa esos archivos y renderiza la página.
Usuario → Navegador → Solicitud HTTP → Servidor
Usuario ← Navegador ← Respuesta HTTP ← Servidor

Archivos comunes de una página

HTML

Define la estructura y el contenido visible de la página.

CSS

Define colores, tamaños, espaciados, tipografías y diseño visual.

JavaScript

Agrega comportamiento, interactividad y lógica en el navegador.

🏗️ HTML (HyperText Markup Language)

HTML es el lenguaje que usamos para escribir páginas de hipertexto mediante etiquetas o tags.

Historia breve

  • 1980: Tim Berners-Lee, investigador del CERN, propuso un sistema de hipertexto para compartir documentos.
  • 1991: HTML fue oficialmente lanzado.
  • Desde entonces, HTML se convirtió en una de las bases principales de internet junto con CSS y JavaScript.

¿Qué necesitas para empezar?

Un editor de texto y tu imaginación es todo lo que necesitas para estar en internet. Para empezar puedes usar VS Code, el navegador y archivos con extensión .html.

¿Qué es una etiqueta?

Una etiqueta describe qué tipo de contenido contiene una parte de la página. Por ejemplo: 

<h1>Título principal</h1>
<p>Este es un párrafo.</p>
<a href="https://developer.mozilla.org/">Leer más</a>

🧱 Estructura HTML

Toda página HTML comienza con una estructura base. Esta estructura indica el tipo de documento, abre el documento HTML, define metadatos y contiene el contenido visible. 

<!DOCTYPE html>
<html>
<head>
  <title>Título de página</title>
</head>
<body>
  Cuerpo de página
</body>
</html>

Partes de la estructura

Elemento Función
<!DOCTYPE html> Declaración del tipo de documento. Le indica al navegador que usaremos HTML moderno.
<html> Apertura de la etiqueta raíz que contiene todo el documento.
<head> Describe metadatos e información importante para navegadores, buscadores, estilos y scripts.
<title> Define el título que aparece en la pestaña del navegador.
<body> Contiene el contenido visible de la página.

⭐ Componentes principales

Los componentes principales son etiquetas que usamos constantemente para estructurar contenido.

Títulos

HTML tiene seis niveles de títulos, desde <h1> hasta <h6>. El <h1> representa el título más importante. 

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección interna</h3>

Párrafos

Los párrafos se escriben con <p> y sirven para bloques de texto. 

<p>HTML nos permite organizar el contenido de una página web.</p>

Enlaces

Los enlaces usan la etiqueta <a> y el atributo href para indicar el destino. 

<a href="https://www.pucetec.edu.ec">Visitar PUCETEC</a>

Imágenes

Las imágenes se agregan con <img>. El atributo src indica la ruta y alt describe la imagen para accesibilidad. 

<img src="assets/logo.png" alt="Logo de PUCETEC">

Campos de entrada

Los campos de entrada son componentes principales porque permiten que el usuario escriba, seleccione o envíe información dentro de una página web. Se crean principalmente con la etiqueta <input>.

Tipo Uso Ejemplo
text Texto corto, como nombres o apellidos. <input type="text" name="nombre">
email Direcciones de correo electrónico. <input type="email" name="correo">
password Contraseñas o datos sensibles que deben ocultarse. <input type="password" name="clave">
number Valores numéricos. <input type="number" name="edad">
date Fechas. <input type="date" name="fecha">
checkbox Opciones que pueden activarse o desactivarse. <input type="checkbox" name="acepta">
radio Selección de una opción dentro de un grupo. <input type="radio" name="turno" value="mañana">
submit Botón para enviar un formulario. <input type="submit" value="Enviar">

Ejemplo de formulario con inputs

<form>
  <label for="nombre">Nombre</label>
  <input id="nombre" name="nombre" type="text">

  <label for="correo">Correo</label>
  <input id="correo" name="correo" type="email">

  <label for="clave">Contraseña</label>
  <input id="clave" name="clave" type="password">

  <input type="submit" value="Enviar">
</form>

🧩 Más componentes principales

Listas

Las listas ayudan a organizar elementos relacionados.

Lista no ordenada

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Lista ordenada

<ol>
  <li>Crear archivo</li>
  <li>Escribir HTML</li>
  <li>Abrir en navegador</li>
</ol>

Contenedores

<div> agrupa bloques de contenido y <span> agrupa texto en línea. 

<div>
  <h2>Perfil</h2>
  <p>Hola, soy <span>Ana</span>.</p>
</div>

🔧 Componentes secundarios

Estos elementos complementan la página y permiten crear contenido más completo.

Tablas

Las tablas organizan información en filas y columnas.

<table>
  <tr>
    <th>Etiqueta</th>
    <th>Uso</th>
  </tr>
  <tr>
    <td>h1</td>
    <td>Título principal</td>
  </tr>
</table>

Formularios

Los formularios permiten agrupar campos de entrada, botones y otros controles para enviar información al servidor. 

<form>
  <label for="nombre">Nombre</label>
  <input id="nombre" name="nombre" type="text">
  <button type="submit">Enviar</button>
</form>

🧰 Más componentes secundarios

Elementos semánticos

HTML moderno incluye etiquetas que comunican el propósito de cada sección de la página.

  • <header>: encabezado de una página o sección.
  • <nav>: navegación principal.
  • <main>: contenido principal.
  • <section>: sección temática.
  • <article>: contenido independiente.
  • <footer>: pie de página.
<header>
  <h1>Mi sitio web</h1>
</header>
<main>
  <section>
    <h2>Noticias</h2>
    <p>Contenido principal de la sección.</p>
  </section>
</main>

Comentarios

Los comentarios sirven para dejar notas en el código. No se muestran visualmente en el navegador. 

<!-- Este es un comentario en HTML -->

🎨 Hojas de Estilos CSS (Cascading Style Sheets)

CSS define la presentación visual de una página web. HTML estructura el contenido; CSS controla cómo se ve.

¿Qué puede cambiar CSS?

  • Colores de texto y fondo.
  • Tamaños y familias de fuente.
  • Espaciados internos y externos.
  • Bordes, sombras y esquinas redondeadas.
  • Distribución de elementos en pantalla.

Ejemplo de CSS

body {
  font-family: Arial, sans-serif;
  background: #f8fafc;
  color: #1f2937;
}

h1 {
  color: #2563eb;
}

Conectar CSS con HTML

Normalmente conectamos una hoja de estilos desde el <head> usando <link>. 

<link rel="stylesheet" href="assets/style.css">

📡 HTTP (Hypertext Transfer Protocol)

HTTP es el protocolo que permite la comunicación entre cliente y servidor en la web.

Cliente y servidor

  • Cliente: Normalmente es el navegador del usuario.
  • Servidor: Equipo o servicio que responde con páginas, archivos o datos.
  • Request: Solicitud enviada por el cliente.
  • Response: Respuesta enviada por el servidor.

Métodos HTTP

Método Uso común
GET Solicitar información o cargar una página.
POST Enviar información nueva al servidor, por ejemplo un formulario.
PUT Actualizar o reemplazar un recurso existente.
DELETE Eliminar un recurso existente.
HEAD Solicitar solo los encabezados de una respuesta, sin el cuerpo. 
GET /index.html HTTP/1.1

❓ Preguntas / Mini desafío

¿Qué significa HTML?

HTML significa HyperText Markup Language, o Lenguaje de Marcado de Hipertexto.

¿Qué parte del HTML contiene el contenido visible?

El contenido visible se escribe dentro de la etiqueta <body>.

¿Para qué sirve CSS?

CSS sirve para definir la apariencia visual de la página: colores, tamaños, fuentes, espacios y layout.

¿Cuál es la diferencia entre GET y POST?

GET se usa para solicitar información. POST se usa para enviar información nueva al servidor.

🎯 Mini desafío final

Desafío: Crea una página HTML que incluya:

  2. Un título principal con <h1>.
  3. Dos párrafos usando <p>.
  4. Una lista de tus sitios web favoritos.
  5. Un enlace a uno de esos sitios.
