Clase: Creación de usuario en Ubuntu y Semántica en HTML

Material de apoyo para estudiantes — desarrollo web.

Objetivo

Aprender a crear un usuario con privilegios en Ubuntu y comprender cómo aplicar semántica en HTML para estructurar correctamente un documento y un formulario.

🧑‍💻 Creación de usuario en Ubuntu

Cada vez que utilices una nueva máquina o sesión de Ubuntu, crea tu usuario personal para mantener tu espacio de trabajo y permisos adecuados.

⚙️ Comando utilizado

sudo /usr/sbin/useradd -G adm,sudo -m -s /bin/bash nombreDeUsuario

🧩 Explicación de cada parte

Parte del comando Significado
sudo Ejecuta el comando con permisos de administrador.
/usr/sbin/useradd Programa que crea un nuevo usuario en el sistema.
-G adm,sudo Asigna al usuario a los grupos adm y sudo, otorgando privilegios administrativos.
-m Crea la carpeta personal del usuario (/home/nombreDeUsuario).
-s /bin/bash Define bash como intérprete por defecto.
nombreDeUsuario Reemplaza con el nombre de usuario que deseas crear.
Ejemplo: 
sudo /usr/sbin/useradd -G adm,sudo -m -s /bin/bash fabian

Luego, establece una contraseña para el usuario creado: sudo passwd fabian

¿Y si el usuario ya existe?

Puedes añadirlo a los grupos necesarios:

sudo usermod -aG adm,sudo nombreDeUsuario

🌐 Semántica en HTML

La semántica en HTML es usar etiquetas por su significado, no solo por su apariencia. Así los navegadores, buscadores y lectores de pantalla comprenden mejor la estructura del documento.

Ejemplos: no semántico vs semántico

No semántico Semántico
<div style="font-size:30px">Formulario</div> <h1>Formulario</h1>
<div>Pie de página</div> <footer>Pie de página</footer>
<b>Nombre:</b> <label for="nombre">Nombre:</label>

Buenas prácticas

  • Usa títulos jerárquicos (<h1>, <h2>, <h3>) en orden.
  • Divide secciones con <section>, <article> y separadores <hr>.
  • Evita usar <div> solo para dar estilo; prefiere etiquetas con significado.
  • Mantén el código limpio, indentado y con atributos bien escritos (name, type, value, id, for).

Mini ejemplo integrando semántica + formulario

<h1>Intereses de Películas</h1>
<hr />
<form action="/buscar" method="get">
  <section>
    <h2>Datos generales</h2>
    <label for="nombres">Nombres:</label>
    <input id="nombres" name="nombres" type="text" required />
  </section>

  <section>
    <h2>Preferencias</h2>
    <p>Género:</p>
    <label><input type="radio" name="genero" value="masculino" /> Masculino</label>
    <label><input type="radio" name="genero" value="femenino" /> Femenino</label>
    <label><input type="radio" name="genero" value="otro" /> Otro</label>

    <p>Categorías de interés:</p>
    <label><input type="checkbox" name="categorias" value="accion" /> Acción</label>
    <label><input type="checkbox" name="categorias" value="comedia" /> Comedia</label>
    <label><input type="checkbox" name="categorias" value="drama" /> Drama</label>
  </section>

  <section>
    <h2>Frecuencia de notificaciones</h2>
    <label for="frecuencia">Cada cuánto deseas recibirlas:</label>
    <select id="frecuencia" name="frecuencia">
      <option value="semanal">Semanal</option>
      <option value="quincenal">Quincenal</option>
      <option value="mensual">Mensual</option>
    </select>
  </section>

  <hr />
  <button type="submit">Enviar formulario</button>
</form>
