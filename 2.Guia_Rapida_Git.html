Guía Rápida: Git + GitLab

Material de apoyo para estudiantes — control de versiones y entrega de tareas.

Objetivo

Aprender el flujo básico para trabajar con Git y GitLab: crear una llave SSH, clonar un repositorio, crear un branch, guardar cambios y subirlos para revisión.

📚 Contenidos

Esta guía resume los comandos mínimos que necesitas para trabajar en tus repositorios de clase.

  • Crear y registrar una llave SSH.
  • Clonar un repositorio desde GitLab.
  • Crear un branch de trabajo.
  • Agregar archivos y crear commits.
  • Configurar nombre y correo en Git.
  • Subir el branch a GitLab.

🔐 1. Crear y registrar tu llave SSH

La llave SSH permite conectar tu computadora con GitLab sin escribir usuario y contraseña en cada operación. Ejecuta este comando en tu terminal: 

ssh-keygen

Presiona Enter en todas las opciones para aceptar la configuración por defecto. Luego, muestra el contenido de la llave pública con: 

cat ~/.ssh/id_rsa.pub

Copia todo el texto de la llave pública y pégalo en GitLab desde Settings → SSH Keys → Add new key.

📥 2. Clonar el repositorio

Una vez configurada la llave, clona el repositorio al que fuiste invitado. El comando crea una copia local del proyecto en tu computadora. 

git clone git@gitlab.com:usuario/nombre-del-repositorio.git

Después de clonar, entra a la carpeta del repositorio:

cd nombre-del-repositorio

🌿 3. Crear y cambiar a tu branch

Un branch es una línea de trabajo independiente. Para tareas de clase, crea un branch con un nombre claro. 

git checkout -b NombreBranch

Ejemplo:

git checkout -b FabianMaqueta

También puedes usar nombres relacionados con la tarea, por ejemplo FabianClaseHTML o FabianFormulario.

💾 4. Agregar y guardar tus archivos

Después de crear o modificar tus archivos, Git necesita saber qué cambios quieres guardar.

Ver el estado del repositorio

git status

Agregar un archivo específico

git add archivo.html

Crear un commit

El commit guarda una versión de tus cambios con un mensaje descriptivo. 

git commit -m "Agrego mi maqueta en HTML"

Configurar nombre y correo

Si Git te pide configurar tu nombre o correo, usa:

git config --global user.name "Tu Nombre"
git config --global user.email "tu_correo@ejemplo.com"

🚀 5. Subir tu branch a GitLab

Finalmente, sube tus cambios al repositorio remoto para que estén disponibles en GitLab. 

git push origin NombreBranch

Ejemplo:

git push origin FabianMaqueta

Después de subir tu branch, GitLab normalmente mostrará una opción para crear un Merge Request. Usa esa opción cuando necesites entregar tu trabajo para revisión.

❓ Preguntas rápidas

¿Qué es Git?

Git es una herramienta de control de versiones que permite guardar el historial de cambios de un proyecto.

¿Qué es GitLab?

GitLab es una plataforma donde puedes alojar repositorios Git, colaborar y entregar cambios mediante Merge Requests.

¿Qué es un branch?

Un branch es una rama de trabajo donde puedes hacer cambios sin modificar directamente la rama principal.

¿Qué es un commit?

Un commit es un punto guardado del historial del proyecto, acompañado por un mensaje que describe el cambio.
