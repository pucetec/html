Guía Rápida: Git + GitHub

Material de apoyo para estudiantes — control de versiones y entrega de tareas.

Objetivo

Aprender el flujo básico para trabajar con Git y GitHub: crear una llave SSH, clonar un repositorio, crear un branch, guardar cambios y subirlos para revisión.

📚 Contenidos

Esta guía resume los comandos mínimos que necesitas para trabajar en tus repositorios de clase.

  • Crear y registrar una llave SSH.
  • Clonar un repositorio desde GitHub.
  • Crear un branch de trabajo.
  • Agregar archivos y crear commits.
  • Configurar nombre y correo en Git.
  • Subir el branch a GitHub.

🔐 1. Crear y registrar tu llave SSH

La llave SSH permite conectar tu computadora con GitHub sin escribir usuario y contraseña en cada operación. Ejecuta este comando en tu terminal: 

ssh-keygen

Presiona Enter en todas las opciones para aceptar la configuración por defecto. Luego, muestra el contenido de la llave pública con: 

cat ~/.ssh/id_rsa.pub

Copia todo el texto de la llave pública y pégalo en GitHub desde Settings → SSH Keys → Add new key.

📥 2. Clonar el repositorio

Una vez configurada la llave, clona el repositorio al que fuiste invitado. El comando crea una copia local del proyecto en tu computadora. 

git clone git@github.com:usuario/nombre-del-repositorio.git

Después de clonar, entra a la carpeta del repositorio:

cd nombre-del-repositorio

🌿 3. Crear y cambiar a tu branch

Un branch es una línea de trabajo independiente. Para tareas de clase, crea un branch con un nombre claro. 

git checkout -b NombreBranch

Ejemplo:

git checkout -b FabianMaqueta

También puedes usar nombres relacionados con la tarea, por ejemplo FabianClaseHTML o FabianFormulario.

💾 4. Agregar y guardar tus archivos

Después de crear o modificar tus archivos, Git necesita saber qué cambios quieres guardar.

Ver el estado del repositorio

git status

Agregar un archivo específico

git add archivo.html

Crear un commit

El commit guarda una versión de tus cambios con un mensaje descriptivo. En clase usaremos siempre este formato: 

git commit -m "fix: [ABC-7204] [Fabian] handle bad request response"

Regla para mensajes de commit:

  • Debe empezar sólo con fix, feat, chore o refactor.
  • Luego va el número de la tarjeta entre corchetes, por ejemplo [ABC-7204].
  • Si no existe una tarjeta, usa [nostory].
  • Después va tu nombre entre corchetes, por ejemplo [Fabian].
  • Finalmente va un mensaje claro y consecuente con los cambios realizados.

¿Qué significa cada opción?

  • fix: se usa cuando corriges un error.
  • feat: se usa cuando agregas una nueva funcionalidad.
  • chore: se usa para cambios de mantenimiento, configuración o tareas internas.
  • refactor: se usa cuando mejoras el código sin cambiar lo que hace.

Configurar nombre y correo

Si Git te pide configurar tu nombre o correo, usa:

git config --global user.name "Tu Nombre"
git config --global user.email "tu_correo@ejemplo.com"

🚀 5. Subir tu branch a GitHub

Finalmente, sube tus cambios al repositorio remoto para que estén disponibles en GitHub. 

git push --set-upstream origin NombreBranch

Ejemplo:

git push --set-upstream origin FabianMaqueta

La opción --set-upstream conecta tu branch local con el branch remoto. Después de hacerlo una vez, normalmente podrás usar sólo git push para los siguientes cambios en ese mismo branch.

Después de subir tu branch, GitHub normalmente mostrará una opción para crear un Pull Request. Usa esa opción cuando necesites entregar tu trabajo para revisión.

Crear un Pull Request

Un Pull Request es la forma de pedir que revisen tus cambios antes de unirlos a la rama principal del proyecto. Después de subir tu branch:

  1. Entra al repositorio en GitHub.
  2. Haz clic en Compare & pull request.
  3. Verifica que tu branch sea el que contiene tus cambios.
  4. Escribe un título claro y una descripción breve de lo que hiciste.
  5. Haz clic en Create pull request.

❓ Preguntas rápidas

¿Qué es Git?

Git es una herramienta de control de versiones que permite guardar el historial de cambios de un proyecto.

¿Qué es GitHub?

GitHub es una plataforma donde puedes alojar repositorios Git, colaborar y entregar cambios mediante Pull Requests.

¿Qué es un branch?

Un branch es una rama de trabajo donde puedes hacer cambios sin modificar directamente la rama principal.

¿Qué es un commit?

Un commit es un punto guardado del historial del proyecto, acompañado por un mensaje que describe el cambio.

🎯 Mini desafío

Desafío: Practica el flujo completo:

  1. Clona un repositorio de prueba.
  2. Crea un branch con tu nombre.
  3. Agrega un archivo HTML simple.
  4. Haz un commit con un mensaje claro.
  5. Sube tu branch a GitHub.
